# Interstellar Spark Odyssey

A browser-based 3D space-exploration experience: lift off from Earth, choose a destination, watch the in-flight transit, and step out onto a Gaussian-splat 3D world generated by [World Labs](https://www.worldlabs.ai/).

Built with:

- [Three.js](https://threejs.org/) for the scene graph and animation loop.
- [Spark.js](https://sparkjs.dev/) (`@sparkjsdev/spark`) for rendering the Gaussian-splat planet surfaces.
- [World Labs Marble API](https://docs.worldlabs.ai/api) to generate the surface worlds (run once, results stored in `src/data/planets.ts`).
- A glassmorphic "Deep Space HUD" interface designed in [Stitch](https://stitch.withgoogle.com/projects/11548640483905168998).

## Destinations (v1)

- **Luna** — Earth's Moon. Gray regolith, Earth in the sky.
- **Mars** — Red dust plains and distant Olympus Mons.
- **Europa** — Jupiter's icy moon. Cracked blue ice sheets.
- **Titan** — Saturn's moon. Orange methane haze.

## Quick start

```bash
npm install

# (option A) seed planets.ts with public Spark sample splats — no API key needed
npm run worlds:mock

# (option B) generate fresh worlds via the World Labs Marble API (~5 min each)
cp .env.example .env.local   # then add your WLT_API_KEY
npm run worlds:generate

npm run dev
```

Open http://localhost:5173.

## Controls

- **Launch Control**: click `INITIATE LAUNCH` to begin.
- **Destination Selector**: click a planet card.
- **In-Flight**: cinematic; `SKIP` button skips the approach.
- **Surface**: click the canvas to capture the pointer, then `WASD` + mouse to walk around. Press `ESC` to release. Click `RETURN TO ORBIT` to pick another planet.

## Scripts

| Script | Purpose |
| --- | --- |
| `npm run dev` | Run the Vite dev server. |
| `npm run build` | Type-check and build for production. |
| `npm run preview` | Preview the production build locally. |
| `npm run worlds:mock` | Populate `src/data/planets.ts` with public sample SPZs (no API key needed). |
| `npm run worlds:generate` | Generate the 4 worlds via the Marble API. Requires `WLT_API_KEY`. |
| `npm run typecheck` | Run TypeScript without emitting. |

## Architecture

```
index.html                      HUD overlays + canvas + import map
src/main.ts                     Renderer + SceneManager + animation loop
src/scenes/SceneManager.ts      State machine: launch -> select -> flight -> arrival -> surface
src/scenes/LaunchScene.ts       Earth + starfield + rocket
src/scenes/FlightScene.ts       Warp starfield + planet approach
src/scenes/SurfaceScene.ts      Spark splat world + first-person controls
src/hud/                        DOM-overlay HUD components per state
src/data/planets.ts             Destination metadata + splat URLs (auto-written)
src/styles/                     Design tokens + HUD styles
scripts/generate-worlds.ts      One-time world generation
```

## Design system

The "Deep Space HUD" tokens live in `src/styles/tokens.css`. Highlights:

- Surface: `#101419` (deep space black)
- Primary: `#00f3ff` (neon plasma cyan)
- Tertiary: `#ffba20` (warning amber)
- Font: Space Grotesk
- Glassmorphism + 1px neon borders + subtle bloom glow
